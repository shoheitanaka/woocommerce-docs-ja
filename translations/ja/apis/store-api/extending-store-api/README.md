# ストアAPIの拡張

アプリケーションは、特定のエンドポイントを拡張することで、Store APIの動作方法を変更できます。特定のエンドポイントにデータを追加することで、サーバーサイドのデータをクライアントサイドで利用できるようになります。

また、Store APIを使用することで、クライアントからサーバーサイドのカート更新をトリガーし、APIから返されたデータでクライアントサイドのカートを更新することもできます。

上記を達成するための詳細については、以下の文書を参照されたい。

| ドキュメント | 説明
|----------|-------------|
| [データを公開する](./extend-store-api-add-data/) | Store APIエンドポイントに追加データを追加する方法を説明します。|
| [利用可能な拡張可能エンドポイント](./available-endpoints-to-extend/) | 拡張可能なすべてのエンドポイントのリスト。|
| [利用可能なフォーマッタ](./extend-store-api-formatters/) | Store APIで使用するデータをフォーマットするための利用可能な`Formatters`。|
| [カートのオンデマンド更新](./extend-store-api-update-cart/) | フロントエンドからのアクションに続いてサーバーサイドのカートを更新します。|
| [フィールドの追加と値の渡し方](./extend-store-api-add-custom-fields/) | Store APIエンドポイントにカスタムフィールドを追加する方法。|
