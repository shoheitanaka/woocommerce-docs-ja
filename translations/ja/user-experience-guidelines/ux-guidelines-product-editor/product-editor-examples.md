# Example Use-Cases

エクステンションのタイプ（およびユースケース）に応じて、ユーザーがどのようにエクステンションを操作するかに最も適したインターフェースの場所を選ぶことができます。

選ぶには、販売者の立場になって考えてみてください。この機能は何に関連していますか？似たような機能のグループにあなたの拡張機能を加えることで、加盟店があなたの拡張機能を見つけやすくなります。

実践的な例については、以下のガイドを参照されたい。

## Product code extension

### What it does

この拡張機能により、加盟店はISBN、EAN、UPCなどの商品識別子を入力できる。

### Our recommendations

識別子は、マーチャントがストアやその他の販売チャネルで商品を説明したり分類したりするのに役立つ1つの情報です。組織グループの商品カタログセクションのフィールドとして追加するのが最適です。

[フィールドの詳細](./product-editor-form-fields.md)

## Ticket extension

### What it does

マーチャントは、独自の入力フィールド、有効期限、制限、階層などの高度な設定でチケットを設定し、販売することができます。

### Our recommendations

多くの高度な機能を持つこのプラグインは、新しい商品タイプを登録し、商品フォームの構造と外観を定義するのがベストでしょう。これには、画面上部のタブ、内部のサブグループとセクション、デフォルト値などが含まれます。

[カスタム商品タイプについての詳細](./product-editor-form-custom-product-types.md)
