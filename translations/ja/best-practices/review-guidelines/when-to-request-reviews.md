# When to request WooCommerce extension reviews

トップスターレビューを増やすための最良のアプローチは、マーチャントがレビューを残す可能性の高い、マーチャント・ジャーニーにおける重要な瞬間を特定し、積極的にレビューを依頼することである。

私たちのユースケースのほとんどで、最も明確な瞬間は次のようなものだ： 

* マーチャントが無力感、迷い、苛立ちを感じているときに、私たちが力になれるとき、
* マーチャントが私たちのコードにバグを発見し、私たちがすぐに修正版を出荷したとき、
* マーチャントが機能を必要としているとき、私たちはそれが出荷されたときに通知します、
* マーチャントが孤独を感じているとき、私たちはその声に耳を傾けています；
* マーチャントが質問をしてきたとき、私たちは一流のサポートを提供するために、たとえ公式のサポート・ポリシーの枠から少し外れることになったとしても、マーチャントに協力します。

誰がレビュー依頼を見るのか、その時何をしているのかを考えましょう。注文を発送しようとしているフルフィルメントワーカーにリクエストを表示しても、うまくいかないでしょう。

節目節目での働きかけはとても効果的です。以前使用した文言は、「xx回目の販売おめでとうございます！WooPaymentsがこのマイルストーンを促進したことを嬉しく思います。WooPaymentsがこのマイルストーンを促進したことを嬉しく思います。

レビューリクエストのタイミングを最適化するもう1つの方法は、使用パターンに合わせてプロンプトを設定することです。例えば、ほとんどのマーチャントが毎日エクステンションを使用していることがわかっている場合、ほとんどのマーチャントがあまり使用しないエクステンションよりも早くレビューリクエストを送信することができます。

SaaS/コネクターのエクステンションは、問題がない限り見落とされる可能性が最も高く、実際のエクステンションを代表しない偏った評価につながるため、正しく評価をリクエストすることに特に注意する必要があります。

特にWordPress.orgのサポートフォーラムでは、サポートの最後にフィードバックをリクエストすることを検討してください。レビューを残す最大の障壁の一つは、ユーザーがWooCommerce.com（またはWordPress.org）にログインしている必要があることであり、WordPress.orgのサポートフォーラムはこれらのレビューを集める良い機会です。公開サポートフォーラムで非常に迅速に対応し、そこで問題を解決することで、ユーザーはすでにログインしており、すぐにレビューを残すことができます（リクエストされた後に！）。
