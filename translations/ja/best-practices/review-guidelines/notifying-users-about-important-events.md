# バグフィックスや機能要望をユーザーに知らせる

バグや不足している機能は、加盟店にとって大きな障害となる可能性があります。バグが積み重なったり、人気のある機能が実装されていなかったりすると、否定的なレビューがついたり、加盟店が競合他社のソリューションに目を向けたりする可能性がある。

バグは通常、サポートやGitHubを通じて報告される。

重大なバグが見つかった場合は、数日以内に解決し（ほとんどの重大なバグは24時間以内に解決する必要があります）、速やかに新しいプラグインバージョンをリリースしてください。リリースプロセスの一環として、すべての関係者（サポートチームと影響を受けるマーチャント）に次のリリースについて通知する必要があります。

たとえ致命的なバグが加盟店にとって大きなストレスの原因であったとしても、迅速な解決は、加盟店の声を聞き、サポートされていると感じさせます。最も困難な状況でも熱心に助けてくれる信頼できるビジネスパートナーがいることは、より強い関係を築くのに役立ちます。そのため、私たちは通常、迅速な解決策を提供した際には、加盟店に5*のレビューをお願いしています。

新しい機能要求を実装し、プラグインの新バージョンを出荷する際にも、バグに対する同様のアプローチをとることができます：

* すべての関係者に通知する。
* 公開アップデートを共有し、「完了」とマークすることで、機能リクエストボードで該当するリクエストを更新する。
* 速報リリースの場合：リリース前にアップデートやニュースレターを発行するために、マーケティング/リレーションチームと連絡を取る。
