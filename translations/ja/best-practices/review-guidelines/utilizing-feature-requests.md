# Utilizing WooCommerce extension feature requests

すべての機能要望を記録し、その製品がどのようなフィードバックを受けているかを誰でも確認できる何らかの記録システムを持つことが重要である。 

毎日または隔日のチェックインをお勧めします：

* 新機能リクエストの選別
* 好意的なレビューを称え
* 否定的なレビューに対処する。

平均的な掲示板には、多くの重複/スパムコンテンツ、実装済みの機能に関するリクエスト、実装されない可能性の高い機能に関するリクエストが含まれているため、機能リクエスト掲示板（または同様のシステム）を注意深く管理することが重要です。メンテナンスが不十分な掲示板は、マーチャントが話を聞いていない／無視されていると感じさせます。

その結果、製品チームが彼らのフィードバックを読んでいなかった／聞いていなかったという前提で、より否定的なレビューが増えることになる。

私たちは、以下の処置で良い結果を得ています：

最も影響を受ける製品から始めて、すべてのオープンリクエストに目を通し、そのほとんど／すべてに返信し、次のように分類する：

* "オープン": より多くのフィードバックが必要なリクエスト、
* 「計画中」は、実装を予定しているリクエストです、
* 「Completed "は、すでに実装されたリクエストです；
* 「クローズ」：製品に適していないため実装を予定していないリクエスト、重複/スパムリクエスト、およびサポートに関する質問です。

すべての "Open "リクエストに返信することが目標ですが、現時点でそれが達成できない場合は、クローズされたリクエストの100％に返信するようにしてください。

機能リクエストとして届いた新しいオープンリクエストについては、掲示板が管理されなくなるのを避けるために、議論/トリアージし、迅速に返信し、ステータスを割り当てる。 

整理整頓されたボードが加盟店に与える効果に加え、製品チームが、どのリクエストが最も求められていて、最も影響力があるのかをよりよく理解し、それに応じて作業計画を立てるのにも役立つ。
